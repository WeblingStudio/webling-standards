# Security Policy

## Supported Versions
Only the latest production release is actively maintained for security fixes.

| Version | Supported |
|---------|-----------|
| Latest  | ✅ |
| Older   | ❌ |


## Reporting a Vulnerability

If you discover a security vulnerability, please do **not** open a public GitHub issue.

Report it privately by emailing: **security@weblingstudio.com**

Include the following in your report:
- A description of the vulnerability and its potential impact
- Steps to reproduce or a proof-of-concept
- Any suggested remediation, if known

You will receive an acknowledgement within **2 business days** and a resolution update within **10 business days**. We will notify you when the issue is resolved.


## Out of Scope
The following are not considered security vulnerabilities for the purposes of this policy:
- Issues in dependencies not directly controlled by this project
- Attacks requiring physical access to a user's device
- Social engineering of Webling Studio staff or clients
- Denial-of-service attacks


## Disclosure Policy
We follow coordinated disclosure. Please allow us to address the vulnerability before disclosing it publicly. We will credit researchers who responsibly disclose valid issues, if they wish.
